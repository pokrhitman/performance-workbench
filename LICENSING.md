# Licensing

Performance Workbench uses two separate licenses, because this repo
contains two genuinely different kinds of work: written educational
content, and functional code. A single license (especially a Creative
Commons license) doesn't fit both well, so they're split deliberately.

**Applies to:** all written material — Explainer and Reference text
across every topic page, the introductory pages, and any other prose.

**What this means in plain terms:**
- You're free to read, share, and link to this content, including for
  teaching or non-commercial use.
- Attribution is required if you share it.
- You may **not** use it commercially without a separate agreement.
- You may **not** publish a modified version of it (translations,
  rewrites, restructured summaries) — share it as-is, or link to it.

https://creativecommons.org/licenses/by-nc-nd/4.0/

## Code — PolyForm Noncommercial License 1.0.0

**Applies to:** everything in `assets/js/`, including the shared formula
engine (`perf-calculations.js`), the CSV import logic, the chart helper,
and the shared nav/tabs/footer scripts.

**What this means in plain terms:**
- You're free to read, run, study, and modify this code for
  non-commercial purposes — personal learning, non-commercial teaching,
  internal non-commercial tools.

Full text: [`LICENSE`](LICENSE) / https://polyformproject.org/licenses/noncommercial/1.0.0/

## Why the split

The written content and the code represent different kinds of work and
carry different risks if reused without permission. Keeping them under
separate, purpose-built licenses protects both clearly, rather than
stretching one license family to cover assets it wasn't designed for.
